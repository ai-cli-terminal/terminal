# v0.3.3

Windows GUI portable packaging patch for the v0.3.x line.

This release keeps the Windows GUI assets introduced in v0.3.2 and fixes the
portable package validation path around optional `WebView2Loader.dll` files in
MSVC Tauri builds.

## What to Download

| Need | Asset |
|---|---|
| Windows GUI installer | `AI.Terminal_0.3.3_x64-setup.exe` |
| Windows GUI portable package | `ai-terminal-windows-x86_64-pc-windows-msvc.zip` |
| Windows CLI helper | `ai-windows-x86_64.exe` |
| Windows shell runtime | `ash-windows-x86_64.exe` |
| Linux CLI helper | `ai-linux-x86_64` |
| Linux shell runtime | `ash-linux-x86_64` |
| Android APK | `ai-terminal-android-universal-unsigned.apk` |

Windows users who want the double-click GUI should download the installer or
portable package and run `ai-terminal.exe`. `ai-windows-x86_64.exe` is the CLI
helper, not the GUI app.

The Android APK is unsigned. It is useful for local/manual testing, but real
release signing and F-Droid buildserver evidence remain follow-up work.

Each binary asset has a matching `.sha256` file. Verify the checksum before
running downloaded binaries.

## Install CLI Assets

Linux or WSL:

```bash
curl -fsSL https://raw.githubusercontent.com/ai-cli-terminal/terminal/main/scripts/install.sh | AI_VERSION=v0.3.3 bash
```

Windows PowerShell:

```powershell
$env:AI_VERSION = 'v0.3.3'
irm https://raw.githubusercontent.com/ai-cli-terminal/terminal/main/scripts/install.ps1 | iex
```

The install scripts install the CLI helper `ai` and shell runtime `ash`. They do
not install the Windows GUI. Use the GUI installer or portable package for that.

## Changes

- Treat `WebView2Loader.dll` as optional in the Windows GUI portable package
  smoke path, matching current MSVC Tauri output.
- Keep the Windows GUI release assets from v0.3.2: portable zip and NSIS
  installer.
- Continue shipping separate `ai` and `ash` binaries for Linux and Windows.
- Continue shipping the unsigned Android universal APK asset.
