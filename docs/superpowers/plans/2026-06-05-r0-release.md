# R0 — 현 상태 릴리즈 (v0.2.0) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 현재 동작하는 `ai` 바이너리를 Linux x86_64 + Windows 네이티브에서 설치·실행 가능한 v0.2.0 배포물(바이너리 + 체크섬 + install 스크립트 + GitHub Release)로 만든다.

**Architecture:** 버전 단일 진실원(`VERSION` ↔ `Cargo.toml` 동기, 테스트로 강제)을 먼저 세우고, feature 매트릭스 빌드를 스크립트화해 검증한 뒤, Windows C-free 빌드를 실증하고, install 스크립트와 태그 기반 릴리즈 CI를 추가한다. 서명은 범위 외(P3-1로 이연), 체크섬(SHA256)까지.

**Tech Stack:** Rust(cargo) · GitHub Actions · bash(install.sh) · PowerShell(install.ps1). feature: `default`/`remote`(C-free, 전 플랫폼) · `storage`(SQLite, C) · `tls`(ring, C).

## 실행 환경 메모 (모든 task 공통)

- **빌드 환경**: Rust 툴체인은 WSL(Ubuntu)에 설치돼 있다(`docs/WORKFLOW.md`). WSL에서 cargo를 돌릴 때의 표준 래퍼:
  ```bash
  wsl.exe -- bash -lc 'source ~/.cargo/env; cd /mnt/c/workspace/act-project/terminal; export CARGO_TARGET_DIR=$HOME/targets/ai-terminal; <cargo 명령>'
  ```
  (현재 체크아웃 경로 `C:\workspace\act-project\terminal` → WSL `/mnt/c/workspace/act-project/terminal`. 다른 머신이면 경로를 맞춘다.)
- **WSL 함정**(`docs/WORKFLOW.md`, `HANDOFF.md §3`): 멀티라인 `bash -lc` 금지(CRLF), 종료코드는 `cmd && echo OK || echo FAIL`로 확인, `git add -A` 금지(명시적 파일 add — 로컬 `.omc/` 오커밋 방지).
- **C-free 우선 원칙**: `default`·`remote`는 C 컴파일러 불필요 → Windows 로컬(C 툴체인 없음)에서도 빌드 가능. `storage`·`tls`는 C 필요 → Linux/WSL 및 MSVC 있는 CI windows-latest에서만.
- 아래 cargo 명령은 표준형으로 적는다. WSL에서 실행 시 위 래퍼로 감싼다.
- 커밋 메시지는 Conventional Commits(`docs/WORKFLOW.md §`) 규칙을 따른다.

- [ ] **Step 1: 빌드 매트릭스 스크립트 작성**

`scripts/build-matrix.sh`:
```bash
#!/usr/bin/env bash
# 릴리즈 feature 매트릭스를 모두 --release 로 빌드한다.
# C-free 조합(default/remote)은 전 플랫폼, C 의존(storage/tls)은 C 툴체인 환경에서만.
# 사용: scripts/build-matrix.sh [--cfree-only]
set -euo pipefail

CFREE=("" "remote")
CDEP=("storage" "tls" "storage tls remote")

run() {
  local feats="$1"
  if [ -z "$feats" ]; then
    echo "==> build --release (default)"
    cargo build --release
  else
    echo "==> build --release --features \"$feats\""
    cargo build --release --features "$feats"
  fi
}

for f in "${CFREE[@]}"; do run "$f"; done

if [ "${1:-}" = "--cfree-only" ]; then
  echo "OK (C-free only)"
  exit 0
fi

for f in "${CDEP[@]}"; do run "$f"; done
echo "OK (full matrix)"
```

```bash
#!/usr/bin/env bash
# AI Terminal 설치 — GitHub Release 에서 Linux x86_64 바이너리를 받아 체크섬 검증 후 설치.
# 사용: curl -fsSL https://raw.githubusercontent.com/ai-cli-terminal/terminal/main/scripts/install.sh | bash
#   환경변수: AI_VERSION(기본 latest), AI_INSTALL_DIR(기본 ~/.local/bin)
set -euo pipefail

REPO="ai-cli-terminal/terminal"
VERSION="${AI_VERSION:-latest}"
INSTALL_DIR="${AI_INSTALL_DIR:-$HOME/.local/bin}"
ASSET="ai-linux-x86_64"

if [ "$VERSION" = "latest" ]; then
  BASE="https://github.com/$REPO/releases/latest/download"
else
  BASE="https://github.com/$REPO/releases/download/$VERSION"
fi

tmp="$(mktemp -d)"
trap 'rm -rf "$tmp"' EXIT

echo "downloading $ASSET ($VERSION)..."
curl -fsSL "$BASE/$ASSET" -o "$tmp/ai"
curl -fsSL "$BASE/$ASSET.sha256" -o "$tmp/ai.sha256"

echo "verifying checksum..."
( cd "$tmp" && sha256sum -c <(printf '%s  ai\n' "$(cut -d' ' -f1 ai.sha256)") )

mkdir -p "$INSTALL_DIR"
install -m 0755 "$tmp/ai" "$INSTALL_DIR/ai"
echo "installed: $INSTALL_DIR/ai"
case ":$PATH:" in
  *":$INSTALL_DIR:"*) ;;
  *) echo "주의: $INSTALL_DIR 가 PATH 에 없습니다. 셸 rc 에 다음을 추가하세요:"; echo "  export PATH=\"$INSTALL_DIR:\$PATH\"" ;;
esac
"$INSTALL_DIR/ai" --version
```

- [ ] **Step 2: Windows install.ps1 작성**

`scripts/install.ps1`:
```powershell
# AI Terminal 설치 (Windows) — GitHub Release 에서 ai.exe 를 받아 SHA256 검증 후 설치.
# 사용: irm https://raw.githubusercontent.com/ai-cli-terminal/terminal/main/scripts/install.ps1 | iex
#   환경변수: AI_VERSION(기본 latest), AI_INSTALL_DIR(기본 $env:LOCALAPPDATA\Programs\ai-terminal)
$ErrorActionPreference = 'Stop'

$repo = 'ai-cli-terminal/terminal'
$version = if ($env:AI_VERSION) { $env:AI_VERSION } else { 'latest' }
$installDir = if ($env:AI_INSTALL_DIR) { $env:AI_INSTALL_DIR } else { Join-Path $env:LOCALAPPDATA 'Programs\ai-terminal' }
$asset = 'ai-windows-x86_64.exe'

$base = if ($version -eq 'latest') {
  "https://github.com/$repo/releases/latest/download"
} else {
  "https://github.com/$repo/releases/download/$version"
}

$tmp = New-Item -ItemType Directory -Path (Join-Path $env:TEMP ([System.Guid]::NewGuid().ToString()))
try {
  Write-Output "downloading $asset ($version)..."
  Invoke-WebRequest "$base/$asset" -OutFile (Join-Path $tmp 'ai.exe')
  Invoke-WebRequest "$base/$asset.sha256" -OutFile (Join-Path $tmp 'ai.exe.sha256')

  Write-Output 'verifying checksum...'
  $expected = (Get-Content (Join-Path $tmp 'ai.exe.sha256') -Raw).Trim().Split(' ')[0].ToLower()
  $actual = (Get-FileHash (Join-Path $tmp 'ai.exe') -Algorithm SHA256).Hash.ToLower()
  if ($expected -ne $actual) { throw "checksum mismatch: expected $expected got $actual" }

  New-Item -ItemType Directory -Force -Path $installDir | Out-Null
  Copy-Item (Join-Path $tmp 'ai.exe') (Join-Path $installDir 'ai.exe') -Force
  Write-Output "installed: $installDir\ai.exe"
  if (-not ($env:Path -split ';' | Where-Object { $_ -eq $installDir })) {
    Write-Output "주의: $installDir 가 PATH 에 없습니다. 다음으로 영구 추가하세요:"
    Write-Output "  setx PATH `"$installDir;`$env:PATH`""
  }
  & (Join-Path $installDir 'ai.exe') --version
} finally {
  Remove-Item -Recurse -Force $tmp
}
```

## Task 5: 태그 기반 릴리즈 CI (R0-5)

`v*` 태그 push 시 ubuntu + windows에서 릴리즈 바이너리를 빌드하고 SHA256 체크섬과 함께 GitHub Release에 업로드한다.

**Files:**
- Create: `.github/workflows/release.yml`

- [ ] **Step 1: 릴리즈 워크플로 작성**

`.github/workflows/release.yml`:
```yaml
name: Release

on:
  push:
    tags: ["v*"]

permissions:
  contents: write

env:
  CARGO_TERM_COLOR: always

jobs:
  build:
    name: build ${{ matrix.os }}
    runs-on: ${{ matrix.os }}
    strategy:
      fail-fast: false
      matrix:
        include:
          - os: ubuntu-latest
            asset: ai-linux-x86_64
            features: "storage tls remote"
            bin: target/release/ai
          - os: windows-latest
            asset: ai-windows-x86_64.exe
            features: "storage remote"
            bin: target/release/ai.exe
    steps:
      - uses: actions/checkout@v4

      - uses: dtolnay/rust-toolchain@stable

      - uses: Swatinem/rust-cache@v2

      - name: Build release
        run: cargo build --release --features "${{ matrix.features }}"

      - name: Package + checksum (Linux)
        if: runner.os == 'Linux'
        run: |
          cp "${{ matrix.bin }}" "${{ matrix.asset }}"
          sha256sum "${{ matrix.asset }}" > "${{ matrix.asset }}.sha256"

      - name: Package + checksum (Windows)
        if: runner.os == 'Windows'
        shell: pwsh
        run: |
          Copy-Item "${{ matrix.bin }}" "${{ matrix.asset }}"
          $h = (Get-FileHash "${{ matrix.asset }}" -Algorithm SHA256).Hash.ToLower()
          "$h  ${{ matrix.asset }}" | Out-File -Encoding ascii "${{ matrix.asset }}.sha256"

      - name: Upload to release
        uses: softprops/action-gh-release@v2
        with:
          files: |
            ${{ matrix.asset }}
            ${{ matrix.asset }}.sha256
```

> Windows feature에서 `tls`를 뺀 이유: `ring`(tls)이 Windows MSVC에서 `nasm`을 요구해 러너 기본 환경에서 실패할 수 있다. 우선 `storage remote`까지 보장하고, `tls`는 후속(nasm 설치 step 추가)으로 둔다 — `default`·`remote` C-free + `storage`는 windows-latest MSVC로 빌드된다.
