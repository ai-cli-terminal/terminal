# Windows GUI Visual Smoke Evidence

Date: 2026-06-28

## Scope

Manual visual inspection of the Windows GUI terminal screenshots generated by
the portable smoke package. This verifies the user-visible product surface of
`ai-terminal.exe` after the automated smoke has launched the packaged Windows
GUI app.

This is not an Explorer double-click recording. It is direct visual inspection
of captured Windows host evidence. If a release gate requires literal Explorer
double-click evidence, run that as a separate manual operator step.

## Command

```powershell
pwsh -NoProfile -ExecutionPolicy Bypass -File .\scripts\smoke-gui.ps1
```

Result:

```text
GUI_SMOKE_OK desktop\src-tauri\target\x86_64-pc-windows-gnu\release\portable\ai-terminal-windows-x86_64-pc-windows-gnu\gui-smoke-evidence.json
```

## Observed Result

The captured screenshots show a self-owned `AI Terminal` GUI window, not
Windows Terminal, PowerShell, cmd.exe, Git Bash, or MSYS. The terminal surface
contains the actual `ash` prompt and command output. The visible transcript
includes:

- `print AI_TERMINAL_GUI_SMOKE_PASTE_OK`
- `ai AI_TERMINAL_GUI_SMOKE_AI_ROUTE`
- `rm -rf /` blocked as Critical
- `cmd.exe /c echo AI_TERMINAL_GUI_SMOKE_EXTERNAL_OK`

The process evidence shows `ash.exe` as the GUI child runtime. No unexpected
external shell descendant is recorded.
